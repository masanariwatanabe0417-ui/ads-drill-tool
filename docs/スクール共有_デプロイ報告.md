## 課題⑤まで完了しました — やったことと、そこから得た結論

**やったこと**

自作の学習ツール（問題のスクショを貼るとAIが解説してくれるツール)で、課題の①〜⑤を完了しました。

- **Neon**（クラウド上のデータベース）にデータを保存するようにした
  → リロードしても、PCを変えても、データが消えなくなった
- **Vercel**（アプリの公開サービス）にデプロイした
  → 公開URLができて、どこからでもブラウザで使えるようになった

**やってみて分かったこと**

講義で「DBが常に正解ではない。今回は学習のためにあえて使う」という話がありましたが、自分のツールで実際にやってみて意味が分かりました。

クラウド向きか、ローカル向きかは、ツールの性質で決まります。

**クラウド（DB＋デプロイ）が向いているもの**
- 複数人で同じデータを使う（予約システム、社内の共有ツールなど）
- スマホや別のPCなど、どこからでも使いたいもの

**ローカルが向いているもの**
- 使うのが自分ひとりだけ
- 自分のPCの中のファイルと直接やりとりするのが便利さの核になっているもの

私のツールは後者でした。「スクショを撮るだけで自動で取り込まれる」のが一番の売りなのですが、これは**自分のPCの中を見張る機能**なので、クラウドのサーバーには持っていけません（クラウドからは他人のPCの中身は見えない＝安全のためにそういう仕組みになっている）。実際、デプロイ版では自動取り込みだけ動きませんでした。

**結論**

デプロイの一連の流れ（DB接続 → 環境変数の設定 → 公開 → 動作確認）は課題として体験できたので、この学びを持って、**今後の開発はローカル版をベースに進めます**。「全部クラウド」「全部ローカル」の二択ではなく、**ツールの性質に合わせて置き場所を選ぶ**のが大事、というのが今回の一番の学びでした。

## 補足メモ（質疑で使える考え方）

**Q. クラウドからPCのファイルに触るのは本当に不可能？**
クラウドのサーバーから直接は不可能（安全装置として意図的に塞がれている）。ただし商品は「ユーザーのPC側に、許可を得て動く相棒を置く」ことで実現している: ①ブラウザに許可をもらう（ファイル選択・ドラッグ&ドロップ・File System Access API） ②常駐アプリを配る（Dropbox方式） ③アプリ全体をインストール型として配る（VS Code方式）。

**Q. 友達に配って、各自のPCで別々に使ってもらうには？**
渡すのはコード（設計図）。データとAPIキーは各自が持つ。レベル1=GitHubごと渡してクローン（要PC知識）、レベル2=インストーラ化して配布（Electron等）、レベル3=APIキー不要のサービス化（課金の仕組みが必要）。今の公開URLを教えるのは×（1つのDBを共有してしまい分離にならない）。
